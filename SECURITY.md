# 🛡️ Security Policy & Responsible Disclosure

This document outlines the security procedures for the **Student Management System**. As a C-based application handling academic and financial data, we prioritize robust binary handling and memory safety.

## 🚀 Supported Versions

| Version | Security Support | Status |
| :--- | :--- | :--- |
| **v1.0.x (Latest)** | Active | :white_check_mark: Supported |
| **< v1.0.0** | None | :x: Not Supported |

## 🔍 Reporting a Vulnerability

Please report security vulnerabilities **responsibly** via email. Do **not** open public issues for security-sensitive bugs.

### 📨 How to Report
Email your report to:
**[mrvishalsinghchauhan01@gmail.com](mailto:mrvishalsinghchauhan01@gmail.com)**

**Please include:**
- Description of the vulnerability (e.g., Buffer Overflow, SQLi).
- Steps to reproduce.
- Impact on the binary data store (`students.dat`).

### ⏱️ Response Timeline
- **Acknowledgment:** Within 48 hours.
- **Triage:** Within 7 days.
